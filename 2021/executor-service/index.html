Java ExecutorService

Java ExecutorService 例子

这个例子创建了一个有十个线程的线程池。

ExecutorService是一个接口，在J.U.C下有两种实现，ThreadPoolExecutor和ScheduledThreadPoolExecutor。

将任务交给线程池执行的方法有

  • execute(Runnable)
  • submit(Runnable)
  • submit(Callable)
  • invokeAny(…)
  • invokeAll(…)

invokeAny()

invokeAny() 方法接受 Callable 集合。 调用这个方法不会返回 Future, 而是其中一个 Callable 的执行结果。 并且不能保证是哪一个 Callable的结果。

只要有一个Callable执行结束, 这个结果就会被 invokeAny()返回, 并且剩余的Callable被取消。

每次执行这段代码打印出来的结果都可能不一样，有时是 “Task 1”, 有时是 “Task 2” 。

invokeAll()

invokeAll() 方法会执行集合中的所有 Callable 对象，然后返回一个Future list

任务抛出异常也是执行结束，Future无法区分正常结束和异常结束。

Runnable vs. Callable

Runnable 代表一个可以被一个线程或者ExecutorService 执行的任务。而Callable只能被ExecutorService执行。另外一个区别是Runnable没有返回值，Callable有返回值。这丛接口声明可以看出来。

取消一个任务

当一个任务还没有被执行的时候，通过调用Future的Cancel()方法可以取消这个任务。

关闭 ExecutorService

为了让线程池中的线程不再执行，当我们使用完ExecutorService后，应该将其关闭。如果应用是通过main（）方法启动的，当主线程退出后，如果线程池中还有活跃的线程，那么这些线程会继续执行，所以我们应该将线程池关闭。

shutdown()

调用shutdown()方法可以关闭线程池，ExecutorService不会立即关闭，但不会接受新的任务，当所有的线程都执行完当前的任务后，ExecutorService关闭。所有在调用shutdown()之前提交的任务都会被执行。

shutdownNow()

如果想要 ExecutorService 立即关闭, 可以调用 shutdownNow() 方法。 这个方法会立即尝试停止所有在执行的任务，并且跳过已提交但未执行的任务。

awaitTermination()

ExecutorService awaitTermination() 会阻塞嗲用这个方法的线程直到 ExecutorService 完全关闭, 或者超过给定的时间。通常在调用 shutdown() 或者 shutdownNow()之后调用awaitTermination() 方法。
